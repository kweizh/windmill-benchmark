# Windmill Workflows-as-Code: Deterministic Timestamp Step (TypeScript) ## Background Windmill's Workflows-as-Code (WAC) runtime uses a checkpoint/replay execution model. Any non-deterministic call placed directly inside the workflow body (such as `new Date().toISOString()`, `Date.now()`, `Math.random()`, or `fetch()`) is unsafe because it will return different values whenever the workflow replays from a checkpoint. The official mitigation is to wrap such calls inside `await step("<key>", () => ...)` so the result is persisted on the first execution and reused on subsequent replays.
