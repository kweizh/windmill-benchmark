### Key design points

| Concern | Approach |
|---|---|
| Bounded concurrency | `{ concurrency: 3 }` option in `parallel()` — never more than 3 child jobs run simultaneously |
| Error resilience | `try/catch` in `checkUrl` maps fetch failures to `{ status: 0, ok: false }` so the workflow always completes |
| Task isolation | Both `checkUrl` and `summarize` are wrapped with `task()` so Windmill tracks them as child jobs |
| Import | Single import from `windmill-client`: `{ task, workflow, parallel }` |
