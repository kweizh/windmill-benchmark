### What was done

#### `data_processor.ts` (main workflow)
- Replaced the `import { task, … }` import with `import { taskScript, workflow }`.
- Removed all three inline helper functions.
- Each `task(localFn)(args)` call was replaced with `taskScript("f/workflows/data_processor/__mod/<name>")(args)`, which schedules the referenced script as a **child job** inside the workflow run.
- The workflow now contains **zero business logic** — it only wires inputs/outputs between steps.

#### `__mod/fetch_data.ts`
- Exports `main(url: string): Promise<any>`.
- Adds a non-2xx guard that was missing in the original.

#### `__mod/process_records.ts`
- Exports `main(data: any[]): Promise<any[]>`.
- Adds an array type-guard for safer error messages.

#### `__mod/save_report.ts`
- Exports `main(results: any[]): Promise<ReportResult>` with an explicit return interface.
- Adds an array type-guard.
