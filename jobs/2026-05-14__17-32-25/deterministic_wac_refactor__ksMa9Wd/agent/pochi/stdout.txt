### What was wrong & why it matters

Windmill's Workflow-as-Code (WAC) runtime replays a workflow's history on each resume/retry. Any value produced by a non-deterministic call (timestamps, random numbers, env reads) **must** be identical across replays; otherwise the workflow diverges from its recorded history and produces incorrect or unpredictable results. Calls made directly in the workflow body run again on every replay and produce different values each time.

The fix is to wrap every such call in `step(() => ...)`. `step()` executes the callback **once**, stores the result in the workflow journal, and returns the stored value on all subsequent replays.
