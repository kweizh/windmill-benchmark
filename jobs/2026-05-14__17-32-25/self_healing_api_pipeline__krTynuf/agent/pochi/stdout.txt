## Architecture overview

```
main(endpoints, max_retries=3)
│
├── builds a list of fetch_with_retry tasks (one per URL)
│
├── parallel(tasks, concurrency=3)   ← Windmill WAC primitive
│     ├── fetch_with_retry(url_1, max_retries)  ┐
│     ├── fetch_with_retry(url_2, max_retries)  ├─ up to 3 at a time
│     └── fetch_with_retry(url_N, max_retries)  ┘
│
├── aggregates results → health report dict
├── writes report to pipeline_report.json
└── returns health report dict
```

---

## Key design decisions

| Requirement | Implementation |
|---|---|
| `endpoints: list`, `max_retries: int = 3` | `async def main(endpoints: list, max_retries: int = 3)` |
| Exponential backoff delays (1 s, 2 s, 4 s…) | `await asyncio.sleep(2 ** (attempt - 1))` inside `@task` |
| **No** wmill sleep primitive inside tasks | `asyncio.sleep()` used exclusively for in-task delays |
| Parallel execution, concurrency cap of 3 | `await parallel(tasks, concurrency=3)` from `wmill` |
| Failed endpoints → `{'status': 'failed', 'data': None}` | Returned explicitly after all retries are exhausted |
| Health report fields | `total`, `successful`, `failed`, `success_rate` (float 0–1) |
| HTTP via stdlib only | `urllib.request.urlopen` + `urllib.error` for exception handling |
| Report persisted to disk | `_write_report()` writes pretty-printed JSON to `pipeline_report.json` |

## Retry / backoff sequence (per endpoint)

```
attempt 1  →  fetch immediately
  ↓ failure
  sleep 2^0 = 1 s
attempt 2  →  fetch
  ↓ failure
  sleep 2^1 = 2 s
attempt 3  →  fetch
  ↓ failure
→ mark as failed  {'endpoint': url, 'status': 'failed', 'data': None}
```
