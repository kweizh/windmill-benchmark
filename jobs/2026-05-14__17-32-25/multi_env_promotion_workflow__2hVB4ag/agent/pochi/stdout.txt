### Step-by-step execution

| # | What happens | Windmill API used |
|---|---|---|
| 1 | Generate a stable, human-readable **promotion ID** (`PROMO-<ts>-<rand>`) | `wmill.step()` |
| 2 | **Deploy → dev** via task `f/tasks/deploy_to_env`; on `success: false` return `{ status: 'failed', stoppedAt: 'dev' }` | `wmill.task()` + early return |
| 3 | **Integration tests on dev** via task `f/tasks/run_integration_tests`; on `passed: false` return failure | `wmill.task()` + early return |
| 4 | **Deploy → staging** in a `try/catch`; on any error → rollback staging → return `{ status: 'failed', stoppedAt: 'staging' }` | `wmill.task()` + try/catch |
| 5 | **Get approval URLs** inside a `step()` and log `urls.approvalPage` | `wmill.step()` + `wmill.getResumeUrls()` |
| 6 | **Wait for approval** (24 h timeout); on rejection/timeout → rollback staging → return `{ status: 'rejected', stoppedAt: 'staging' }` | `wmill.waitForApproval({ timeout: 86400 })` |
| 7 | **Deploy → production** in a `try/catch`; on any error → rollback **both** production and staging in parallel → return `{ status: 'failed', stoppedAt: 'production' }` | `wmill.task()` + `Promise.all(rollbacks)` |
| 8 | Return `{ promotionId, artifactId, version, status: 'promoted', environments: ['dev','staging','production'] }` | — |

---

### Key design decisions

* **No rollback on dev failure** — dev is ephemeral; only staging and production need rollback.
* **try/catch on staging & production** — catches both task exceptions and logical `success: false` results (re-thrown as `Error`).
* **Parallel rollback on production failure** — `Promise.all([rollbackEnv('production'), rollbackEnv('staging')])` minimises blast radius time.
* **`approvalUrls` is returned from the `step()`** — makes the URL available in Windmill's step output panel without needing a separate variable.
* **Rejection returns exactly** `{ promotionId, status: 'rejected', stoppedAt: 'staging' }` as specified (plus `artifactId`/`version` for traceability).
