### Architecture overview

```
main(events, warehouseTarget)
│
├─ step("read_processed_ids")          ← persists result for replay-safety
│   └─ readProcessedIds()             ← reads processed_events.json
│
├─ [inline] deduplicate               ← Set-based filter; early-return if 0 new events
│
├─ parallel(newEvents, enrichEvent, { concurrency: 5 })
│   └─ task(enrichEvent)(event) × N   ← each event → separate child job, ≤5 at once
│
├─ step("get_approval_urls")
│   └─ getResumeUrls()                ← cached URL stable across replays
│   └─ console.log(approvalPage, resume, cancel)
│
├─ waitForApproval({ timeout: 86400 })
│   └─ worker released, 0 resource usage while suspended
│
├─ if (!approval.approved)
│   └─ return rejection summary       ← no warehouse write, no state update
│
├─ task(writeToWarehouse)(enrichedEvents, warehouseTarget)
│
├─ step("update_processed_ids")
│   └─ merge old + new IDs → writeProcessedIds()  ← atomic, idempotent on replay
│
└─ return success summary
```

### How each requirement is satisfied

| Requirement | Implementation |
|---|---|
| `events` + optional `warehouseTarget` params | Typed parameters on the `workflow()` entry point |
| Deduplicate via `processed_events.json` | `step("read_processed_ids")` reads the JSON; `Set`-based filter removes already-seen IDs |
| Enrich new events in parallel, max 5 | `parallel(newEvents, (e) => task(enrichEvent)(e), { concurrency: 5 })` |
| Obtain & log resume/approval URLs | `step("get_approval_urls", () => getResumeUrls())` + `console.log` of `approvalPage`, `resume`, `cancel` |
| Suspend for human approval | `waitForApproval({ timeout: 86400 })` — zero worker usage while waiting |
| On approval → write warehouse + update state | `task(writeToWarehouse)` then `step("update_processed_ids")` merges IDs and flushes JSON |
| On rejection → rejection summary, no write | `if (!approval.approved)` branch returns summary without touching state file |

---

### Windmill WAC primitives used

- **`workflow()`** — marks the entry point; enables `task`, `step`, `parallel`, `waitForApproval`
- **`step(name, fn)`** — persists lightweight results (file read, URL generation, file write) so replays return cached values without side-effects
- **`task(fn)`** — runs `enrichEvent` and `writeToWarehouse` as isolated child jobs with their own logs and worker slots
- **`parallel(items, fn, { concurrency })`** — fans out enrichment across all new events, bounded to 5 concurrent child jobs
- **`getResumeUrls()`** — generates the `approvalPage` / `resume` / `cancel` URL triple
- **`waitForApproval({ timeout })`** — suspends the workflow (releasing the worker) until a human acts; returns `{ approved, approver, value }`
