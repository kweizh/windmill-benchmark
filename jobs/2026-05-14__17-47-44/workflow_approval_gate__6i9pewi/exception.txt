## Background

Windmill's `waitForApproval()` primitive suspends a workflow until a human explicitly approves or rejects it via the Windmill UI. The worker is fully released while waiting, consuming zero resources. It returns `{ approved: boolean, approver: string, value: any }`.

## Requirements

1. Defines a `runTests` task function that accepts `branch: string` and returns `{ branch, tests_passed: true, coverage: 98 }`.
2. Defines a `deployToProd` task function that accepts `branch: string` and returns `{ branch, status: 'deployed', url: 'https://prod.example.com' }`.
3. Exports `main` as a `workflow()` that:
   - Accepts `branch: string` as input.
