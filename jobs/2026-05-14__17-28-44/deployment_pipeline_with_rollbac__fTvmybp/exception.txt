# Build a Windmill Deployment Pipeline with Rollback and Human Approval Gate

## Background

Windmill WAC (Workflows-as-Code) supports robust deployment pipelines through a combination of primitives:

- `step()` — wraps a non-deterministic call so its value is computed once and cached across workflow replays. Used to generate a stable deployment ID.
- `task()` — schedules a function as an isolated task step, enabling retries and distributed execution.
- `waitForApproval()` — suspends the workflow until a human approves or rejects it via the Windmill UI, returning `{ approved: boolean, approver: string, value: any }`.
- `try/catch` — standard error handling; on task failure, a rollback task can be called before rethrowing.

Combining these primitives gives you a production-grade deployment pipeline: generate a stable deploy ID, run tests, deploy to staging (with rollback on failure), wait for human sign-off (with rollback on rejection), deploy to production (with rollback on failure), and return a success summary.

## Requirements

Create a TypeScript workflow at `/home/user/windmill-project/f/workflows/deploy_with_rollback.ts` that:

1. **Imports** `{ task, workflow, step, waitForApproval }` from `windmill-client`.
2. **Imports** `crypto` from `node:crypto`.
3. **Defines** the following plain async functions (used as task implementations):
   - `runTests(branch: string)` — returns `{ branch, passed: true, tests: 42 }`
   - `deployToStaging(branch: string, deployId: string)` — returns `{ deployId, env: 'staging', url: 'https://staging.example.com' }`
   - `rollbackStaging(deployId: string)` — returns `{ deployId, rolled_back: 'staging' }`
   - `deployToProduction(branch: string, deployId: string)` — returns `{ deployId, env: 'production', url: 'https://prod.example.com' }`
   - `rollbackProduction(deployId: string)` — returns `{ deployId, rolled_back: 'production' }`
4. **Exports** `main` as a `workflow()` that accepts `branch: string` and:
   a. Captures a stable deploy ID: `const deployId = await step('deploy_id', () => crypto.randomUUID());`
   b. Calls `const testResult = await task(runTests)(branch);` — if `testResult.passed === false`, throws `new Error('Tests failed')` immediately.
   c. Wraps `await task(deployToStaging)(branch, deployId)` in a `try/catch`; on catch, calls `await task(rollbackStaging)(deployId)` and rethrows.
   d. Calls `const approval = await waitForApproval({ timeout: 7200 });` — if `!approval.approved`, calls `await task(rollbackStaging)(deployId)` and returns `{ deployId, status: 'rejected', branch, approver: approval.approver }`.
   e. Wraps `await task(deployToProduction)(branch, deployId)` in a `try/catch`; on catch, calls `await task(rollbackProduction)(deployId)` and rethrows.
   f. Returns `{ deployId, status: 'deployed', branch, approver: approval.approver }`.

2. Add the required imports at the top:

3. Implement the five task functions (no decorator needed — they are plain async functions passed to `task()`).
4. Implement `main` using `workflow()`. A full skeleton:
   ```typescript
   export const main = workflow('deploy_with_rollback', async (branch: string) => {
     const deployId = await step('deploy_id', () => crypto.randomUUID());

     const testResult = await task(runTests)(branch);
     if (!testResult.passed) {
       throw new Error('Tests failed');
     }

     try {
       await task(deployToStaging)(branch, deployId);
     } catch (e) {
       await task(rollbackStaging)(deployId);
       throw e;
     }

     const approval = await waitForApproval({ timeout: 7200 });
     if (!approval.approved) {
       await task(rollbackStaging)(deployId);
       return { deployId, status: 'rejected', branch, approver: approval.approver };
     }

     try {
       await task(deployToProduction)(branch, deployId);
     } catch (e) {
       await task(rollbackProduction)(deployId);
       throw e;
     }

     return { deployId, status: 'deployed', branch, approver: approval.approver };
   });
   ```
5. Save the file.
