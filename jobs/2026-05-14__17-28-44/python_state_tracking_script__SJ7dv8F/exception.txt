# Windmill Python State Tracking Script

## Background
Windmill is a developer platform for building internal tools and workflows. Python scripts in Windmill export a `main()` function and can persist state across executions using the `wmill` client library. The `get_state()` function retrieves the previously stored state (returns `None` on the very first run), and `set_state(value)` persists a new value for the next run.

You have a Windmill project scaffolded at `/home/user/windmill-project`. Your task is to create a Python script that acts as an execution counter — every time it runs, it reads the previous count, increments it, saves the new count, and returns a summary.

## Requirements
- Write a Python Windmill script at `/home/user/windmill-project/f/scripts/execution_counter.py`.
- The script must import `get_state` and `set_state` from the `wmill` package.
- Define a `main()` function (no parameters required).
- Inside `main()`, read the current count from state using `get_state()`; if the value is `None` (first run), treat it as `0`.
- Increment the count by `1`.
- Persist the new count using `set_state(new_count)`.
- Return a dict: `{"count": new_count, "message": f"This script has been executed {new_count} time(s)."}`.
- Also create a companion metadata file at `/home/user/windmill-project/f/scripts/execution_counter.script.yaml` with the following content:

```yaml
description: Tracks the number of times this script has been executed using Windmill persistent state.
language: python3
```

## Implementation Guide

   ```bash
   mkdir -p /home/user/windmill-project/f/scripts
   ```
2. Write the script file `/home/user/windmill-project/f/scripts/execution_counter.py`:
   ```python
   from wmill import get_state, set_state

   def main():
       current_count = get_state()
       if current_count is None:
           current_count = 0

       set_state(new_count)
       return {
           "count": new_count,
           "message": f"This script has been executed {new_count} time(s)."
       }
   ```
