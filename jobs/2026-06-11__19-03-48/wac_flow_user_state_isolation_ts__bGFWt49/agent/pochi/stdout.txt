Unlike `getState` / `setState` (which is global to a script asset across runs), the user state lives on the root flow/job for the duration of a single execution and is automatically isolated between independent runs. You will author a TypeScript WAC workflow that uses these calls to accumulate the integers from an input array into a single running total across multiple steps within one execution, then return the final value.
