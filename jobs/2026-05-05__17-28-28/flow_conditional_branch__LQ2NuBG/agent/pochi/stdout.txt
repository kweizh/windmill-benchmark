## Flow Logic

```
[a] check_score (score=75 static)
       │
       ▼
[b] branchone
   ├── branch: results.a.passed === true  →  [c] on_pass(score=results.a.score)
   └── default                            →  [d] on_fail(score=results.a.score)
```

- Module **a** runs `check_score` with a hardcoded static score of `75`, returning `{ score: 75, passed: true }`.
- Module **b** is a `branchone` that evaluates `results.a.passed === true`. Since `75 >= 60`, the **Passed** branch fires, executing module **c** (`on_pass`) and yielding `"Score 75: PASSED"`.
- If the score were below 60, the default branch would execute module **d** (`on_fail`), yielding `"Score N: FAILED"`.
- Both branch modules receive `score` via a `javascript` input transform (`results.a.score`), ensuring the value flows from the first step.
