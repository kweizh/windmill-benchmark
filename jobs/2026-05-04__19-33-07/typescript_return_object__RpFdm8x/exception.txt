# TypeScript Windmill Script Returning a Structured Object

## Background

Windmill scripts can return structured objects, not just primitives. The returned object is serialized to JSON and made available to downstream flow steps or displayed in the Windmill UI. This is useful for scripts that produce multiple named output values.

## Requirements

- Create a TypeScript script at `/home/user/windmill-project/f/scripts/user_info.ts`.
- The script must export an async `main` function with this signature:
  ```typescript
  export async function main(id: number, name: string, active: boolean = true)
  ```
- The function must return an object with exactly these fields:
  ```typescript
  { id: number, name: string, active: boolean, display: string }
  ```
  Where `display` is the string `` `[${id}] ${name} (${active ? 'active' : 'inactive'})` ``.
- Create the metadata file at `/home/user/windmill-project/f/scripts/user_info.script.yaml` with:
  - `summary: "Build a user info object"`
  - `language: typescript`

## Implementation Guide

1. Create `f/scripts/user_info.ts`:
   ```typescript
   export async function main(id: number, name: string, active: boolean = true) {
     return {
       id,
       name,
       active,
       display: `[${id}] ${name} (${active ? 'active' : 'inactive'})`,
     };
   }
   ```
2. Create `f/scripts/user_info.script.yaml`:
   ```yaml
   summary: "Build a user info object"
   language: typescript
   ```
