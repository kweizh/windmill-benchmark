### What was built

**`flow.yaml`** follows the OpenFlow spec with three key sections:

| Section | Purpose |
|---|---|
| `schema` | Declares the single boolean input `should_fail` (required) |
| `value.modules[0]` (`main_step`) | Python rawscript — raises `Exception` when `should_fail` is `true`, otherwise returns `{"ok": True}` |
| `value.failure_module` (`error_handler`) | Python rawscript — intercepts the failure via the built-in `error` variable and returns `{"handled": True, "reason": "<error message>"}` |

**Error handler input transform** uses `expr: error` — the `error` object Windmill injects into the failure module automatically contains a `"message"` field with the upstream exception text, which is forwarded as the `reason` field.
